## はじめに

皆さんは自分のこれまでのアウトプット（プルリクエスト）を振り返りたい時、どうしていますか？
GitHubのプロフィールでも直近のアクティビティを確認できますが、「もっと自分好みに、一覧性高く見たい」という思いから、自分専用のPR一覧サイトを制作しました。
制作のきっかけは、ryoppippiさんとKanonさんのポストで、自分自身のPRを美しくまとめたサイトを拝見したことです。

https://x.com/ryoppippi/status/1826204396486611394

https://x.com/ysknsid25/status/1997309774682345571

「自分の活動が可視化されるの、めちゃくちゃいいな…！」と衝撃を受け、自分も独自の機能を盛り込んで作ってみることにしました。

## なぜ「PR一覧サイト」を作るのがおすすめなのか？

これから個人開発を始めたい人や、技術の練習をしたい人にこのネタは非常におすすめです。
理由は3つあります。

- モチベーションが明確

「自分のPRを一覧で見たい」という実用的な目的があるため、完成まで走りきりやすいです。
公式には意外と「自分専用の一覧ページ」はないので、作る価値があります。

- 構成がシンプルながら学びが多い

基本は「APIからデータを取ってきて表示する」というシンプルな作りです。
一方で、GitHub APIのトークン（Credential）を扱うため、フロントエンドだけでなくバックエンドの実装も求められます。
デプロイ先や、どのようにデータを更新するかも考えなければならないため、意外と考えることは多いです。

- コントリビューションが捗る

「このサイトを自分の実績で埋め尽くしたい！」という気持ちになり、OSS活動や開発のモチベーションが勝手に上がります。
これが一番大きい。

## 私がこだわった工夫ポイント

![サイト](/images/my-github-prlist/image.png)

https://pr.bmth.dev/

Kanonさんの構成を参考にしつつ、より「ダッシュボード」としての使い勝手を追求して以下の機能を実装しました。

1. 検索機能を強化
PRタイトルやリポジトリ名での部分一致検索を実装しました。
これらの状態管理にContextを使いたくなかったため、nuqsを使ったクエリパラメータで実装しています。
これにより、リロードしても検索条件が維持されます。

2. Organizationによる絞り込み機能
「特定の組織のPRだけ見たい」というケースが多いと思ったので、Organizationごとのフィルタリングを実装しました。
画面を行ったり来たりせず、一画面で完結するダッシュボード風UIを採用しています。

3. レスポンシブ対応
PCでじっくり眺めるのはもちろん、スマホからでもサクッと進捗を確認できるよう、それぞれのデバイスに最適化したレイアウトを組みました。

4. Yamada UIの採用
見ての通り、Yamada UIのメンテナーをしているので、今回のサイトでも採用しています。
シンプルな作りにしてしまうと、Yamada UIを活かせないということもあって、このようなデザインを採用した次第です。
特に、Tableコンポーネントを使っている箇所は、Yamada UIの良さが出ています。
ここはぜひ実装を見ていただきたいです。
